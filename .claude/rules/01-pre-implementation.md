# Pre-Implementation Analysis

Before writing any code, create a brief analysis that includes:

1. **Requirements** - List explicit and implicit requirements from the request
2. **Assumptions** - Document any assumptions being made
3. **Constraints** - Identify technical or business limitations
4. **Architecture** - Propose module structure and data flow
5. **Potential issues** - Flag redundancies, anti-patterns, or edge cases

All proposed changes must include reasoning and expected benefits (clarity, efficiency, maintainability, testability).

Do not start coding until the analysis is reviewed or acknowledged.
